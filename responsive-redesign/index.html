Introduction

Not too long ago, visiting triolosbakery.com in your web browser would have taken you to the website for Triolos Bakery, a small bakery in the town of Bedford NH. But there was just one problem: the website was built by a former UNIX developer who put it together using the rudimentary knowledge of HTML and CSS that he picked up in the nineties. And it looked the part. For a class assignment in CSCI 1300: User Interfaces / User Experiences at Brown University, I designed a responsive replacement for the website based on my evaluation of its usability.

Out with the old...

triolosbakery.com (circa 2019)

Problems

I identified five problems with the old website.

1

The old website was not responsive. In particular, the navigation menu and center image did not adapt to a tablet or mobile screen.

Key concepts

Responsive, Hierarchy

2

The old website drew the attention of a new visitor to the centered image of the bakery without doing anything with that attention.

Key concepts

Learnability

3

The navigation menu offered too many options to choose from, including the secondary items that appeared upon hover.

Key concepts

Memorability, Learnability

4

The old website had a lot of empty space without a purpose, especially the space around the centered image of the bakery

Key concepts

Efficiency, Layout

5

The old website effectively hid the contact information from new visitors by including it at the bottom and using a tiny font size.

Key concepts

Learnability, Efficiency

Accessibility

I also entered the old website into the WAVE tool to evaluate it in terms of its accessibility. On the one hand, the website had a few problems: it did not specify that it used the English language; it had an empty link that went nowhere; a few links were duplicates; and the copyright at the text was too small for users to see. On the other hand, many of its images had alternative text available, and it made use of a structured navigation menu with several tiers.

In with the new!
